## 정의

브랜치 목적에 따라 머지 방식을 다르게 적용해 커밋 그래프의 가독성과 이력의 명확성을 유지하는 팀 규칙.

## 작동 원리

| 머지 방향 | 방식 | 이유 |
|-----------|------|------|
| 기능 브랜치 → `dev` | Squash and Merge | 작업 중 오타·리뷰 반영 커밋을 하나로 정리 |
| `release` → `main` | Merge Commit | 배포 단위 이력을 명확하게 남기기 위해 |
| `release` → `dev` | Rebase and Merge | 불필요한 머지 커밋 없이 수정 사항만 반영 |

## 적용 사례

```
feat/KNK-12 → dev         : Squash and Merge
release/v1.0.0 → main     : Merge Commit
release/v1.0.0 → dev      : Rebase and Merge
```
