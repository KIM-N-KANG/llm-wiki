## 정의

Pull Request 제목 형식, 본문 작성 기준, 머지 방식을 규정한 팀 PR 운영 규칙.

## 작동 원리

**제목 형식:**
```
[KNK-{Jira이슈번호}] {태그}: {PR 제목}
```

**본문 작성 기준:**
- 작업 내용 구체적으로 기술
- UI 작업 포함 시 화면 캡처 첨부
- 테스트 수행 시 테스트 방법과 결과 함께 작성

**태그 목록:**

| 태그 | 사용 기준 |
|------|----------|
| `Feat` | 새로운 기능 추가 |
| `Fix` | 버그 수정 |
| `Docs` | 문서 수정 |
| `Design` | UI·스타일·디자인 수정 |
| `CICD` | 배포·CI/CD 설정 수정 |
| `Refactor` | 기능 변화 없는 코드 구조 개선 |
| `Chore` | 설정·의존성·기타 유지보수 |
| `Release` | 새로운 버전 배포 |

## 적용 사례

```
[KNK-1] Feat: 멤버 목록 화면 UI 구현
[KNK-12] Fix: 로그인 페이지 오버플로우 수정
[KNK-45] Release: v1.1.0 배포
```
