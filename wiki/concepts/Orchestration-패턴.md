## 정의

AI에게 일을 시키는 방식. 단일 에이전트, 부하 파견(Subagent), 팀 협업(Team Mode) 세 가지 패턴이 있으며, 상황에 맞는 패턴을 선택해야 한다. [[Harness-Engineering]]의 6개 축 중 "실행(Orchestration)" 축에 해당한다.

## 작동 원리

**3가지 실행 패턴**:

| 패턴 | 구조 | 적합 상황 | 비용 |
|------|------|-----------|------|
| 혼자·Single | AI → 결과물 | 단순 작업, 대부분의 일상 | 기준 |
| 부하 파견·Subagent | 메인 → (조사/작업/검증) → 결과 종합 | 병렬/전문화, 위임 → 종합 | 중간 |
| 팀 협업·Team Mode | PM/개발/디자인/QA 에이전트 간 소통 | 다관점·복잡·에이전트 간 소통 필요 | ~7x |

> 90%는 단일/서브에이전트로 충분. Team Mode는 토큰 비용이 ~7배.

**상황별 오케스트레이션 패턴**:
- **순차 파이프라인**: 조사 → 초안 → 퇴고 → 발행처럼 순서가 중요한 경우. TaskCreate로 체크박스 생성, 하나씩 직렬 수행
- **병렬 Subagent**: 경쟁사 3곳의 랜딩 페이지를 동시에 분석처럼 서로 독립적인 경우. Agent 3개 spawn, 병렬 분석 후 메인이 취합
- **Team Mode**: 새 기능을 설계하면서 동시에 구현하고 리뷰도 받아야 하는 경우. TeamCreate로 팀 생성, 에이전트 간 직접 소통

### Ralph Loop
완료 기준을 합의하고, 그 기준을 충족할 때까지 AI가 알아서 반복하는 패턴.
1. 완료 기준 합의 ("모바일 반응형 + Lighthouse 90점 이상 + 카피 3번 이상 퇴고")
2. AI가 작업
3. 기준 충족? → NO: 2번으로 돌아가기 / YES: 완료
> "뭐가 되면 끝인지"만 정해주면 AI가 될 때까지 계속 돈다

**Auto Research (karpathy/autoresearch)**:
사람은 방향(program.md)만 정하고, AI가 밤새 실험을 돌리는 자율 실험 루프.
- AI가 train.py만 수정 (아키텍처, 하이퍼파라미터 변경)
- 시간당 ~12개 실험 자율 수행, 밤새 무인 운영
- 성능 개선되면 유지, 아니면 폐기 → 다음 실험

## 적용 사례

- 블로그 글 작성: "AI 에이전트 트렌드 조사 → 초안 작성 → 퇴고까지 순서대로 진행해줘" (순차)
- 경쟁사 분석: "A사, B사, C사 랜딩 페이지를 각각 에이전트 파견해서 동시에 분석해줘. 끝나면 비교표 만들어" (병렬)
- 기능 설계+구현: "Team 커서 설계자, 구현자, 리뷰어 3명으로 팀 꾸려줘. 설계 나오면 구현자가 바로 시작해" (팀)
