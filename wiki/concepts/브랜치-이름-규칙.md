## 정의

작업 브랜치 이름을 태그, Jira 이슈 번호, 작업 제목으로 구성해 브랜치 목적을 즉시 파악할 수 있게 하는 네이밍 규칙.

## 작동 원리

**형식:**
```
{태그}/{KNK-이슈번호}-{브랜치_제목}
```

**구분자 규칙:**
- 태그와 이슈 번호 사이: `/`
- 이슈 번호와 제목 사이: `-`
- 제목 단어 사이: `-`
- 태그는 소문자

**태그 목록:**

| 태그 | 사용 기준 |
|------|----------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 |
| `design` | UI·스타일·디자인 수정 |
| `cicd` | 배포·CI/CD 설정 수정 |
| `refactor` | 기능 변화 없는 코드 구조 개선 |
| `chore` | 설정·의존성·기타 유지보수 |

## 적용 사례

```
feat/KNK-1-member-list-page
fix/KNK-12-overflow-in-login-page
```
